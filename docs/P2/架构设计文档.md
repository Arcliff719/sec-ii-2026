

## 项目技术栈
| 层次 | 选择 | 选择理由 |
|------|------|---------|
| 前端框架 | Vue 3 | 学习曲线平缓，SFC 模板语法接近原生 HTML/CSS/JS，零基础团队可快速上手；中文生态完善，文档质量高；渐进式设计，可按需引入状态管理和路由 |
| 后端框架 | Spring Boot (Java) | 团队全体有 Java 课程基础，无需从零学习新语言；类型安全 + 枚举适合订单状态机的严格转换规则；Spring Security + JWT 认证体系成熟；声明式事务保障数据一致性 |
| 数据库 | MySQL 8.0 | 关系型数据，事务支持完善，与 JPA 集成成熟，团队课程中有使用经验 |
| 缓存 | Redis | 用于 JWT 会话管理和热点数据缓存，与 Spring Boot 集成成熟 |
| 部署方式 | 优先桌面端 Web，后续考虑移动端适配和小程序 | MVP 阶段桌面端优先，Element Plus 以桌面端设计为主；后续若有移动端需求再评估 Vant UI 或 UniApp |

## 架构图

```
┌──────────────────────────────────────┐
│           展示层 (Vue 3)              │
│  Element Plus + Pinia + Vue Router   │
│            (独立前端项目)             │
└──────────────┬───────────────────────┘
               │ RESTful API (JSON over HTTP)
               │ JWT Token 认证
┌──────────────▼───────────────────────┐
│         Controller 层                │
│   Spring MVC (@RestController)       │
│   请求参数校验、响应格式化             │
└──────────────┬───────────────────────┘
               │
┌──────────────▼───────────────────────┐
│          Service 层                  │
│   Spring @Service + @Transactional   │
│   订单状态机、信用积分计算             │
└──────────────┬───────────────────────┘
               │
┌──────────────▼───────────────────────┐
│        Repository 层                 │
│   Spring Data JPA                    │
│   数据访问，方法命名约定自动生成查询    │
└──────────────┬───────────────────────┘
               │
┌──────────────▼───────────────────────┐
│         数据库 / 缓存层               │
│   MySQL 8.0 + Redis                  │
└──────────────────────────────────────┘
```

### 分层纪律：
- Controller 层仅负责请求参数校验和响应格式化，不包含业务逻辑；
- Service 层集中实现所有业务规则；
- Repository 层仅负责数据存取；
- 禁止跨层调用。

## 模块划分

| 模块 | 功能描述 | 涉及技术点 |
|------|---------|-----------|
| 用户管理 | 注册/登录、JWT 认证、个人资料管理、校内身份验证 | Spring Security + JWT、Vue 表单 |
| 需求发布与浏览 | 需求分类、发布、列表检索、详情查看 | JPA 动态查询、Element Plus 表格/表单/对话框 |
| 订单流程管理 | 状态机流转（待接单 → 进行中 → 待评价 → 已完成/已取消） | Java Enum 状态机、@Transactional 声明式事务 |
| 评价与信用体系 | 双向评价评分、信用积分计算与展示 | Service 层业务规则、Element Plus 评分组件 |
| 消息通知 | 订单状态变更通知、系统消息推送 | 后续可从 Service 层拆分为独立服务 |
