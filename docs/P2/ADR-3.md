
# ADR-3：后端语言选择 Java

## 状态
已接受

## 背景

后端需要实现以下核心业务：用户注册登录与校内认证、需求发布与分类检索、订单状态机（待接单 → 进行中 → 待评价 → 已完成/已取消）、双向评价与信用积分计算、消息通知。
团队 4 人均在课程中学习过基础Java内容，具有一定的编程基础，但是缺少项目经验。候选方案为 Python（Django REST Framework）和 Java（Spring Boot）。

## 决策

采用 **Java 17 + Spring Boot 3** 作为后端技术栈。

技术栈细节：
- 框架：Spring Boot 3 + Spring MVC
- ORM：Spring Data JPA（Hibernate）
- 安全：Spring Security + JWT
- 数据库：MySQL 8.0
- 缓存：Redis（会话管理 + 热点数据）
- 构建：Maven
- API 文档：SpringDoc（OpenAPI 3.0）

## 理由

1. **团队技能匹配**：全体成员在课程中学习过Java相关的基础内容，无需从零开始学习新语言；已有的 Java 基础可直接转化为后端开发能力，缩短学习爬坡期
2. **类型安全适合复杂业务**：订单状态机涉及严格的状态转换规则（待接单 → 进行中 → 待评价 → 已完成/已取消），Java 的静态类型 + 枚举（Enum）可在编译期捕获状态非法转换，减少运行时错误
3. **Spring Boot 生态成熟**：
   - Spring Security 提供开箱即用的认证授权体系，JWT 集成有成熟的最佳实践
   - Spring Data JPA 通过方法命名约定自动生成查询，减少手写 SQL
   - 声明式事务管理（`@Transactional`）保障订单状态变更的数据一致性
4. **企业级特性开箱即用**：连接池（HikariCP）、Bean Validation、全局异常处理、Actuator 健康检查——这些是生产环境必需的，Spring Boot 均已内置
5. **教学资源丰富**：Spring Boot 中文教程、CSDN 社区覆盖全面，遇到问题有大量可参考的中文资料
6. **为什么不选 Python/Django**：虽然 Django 自带 Admin 后台和 ORM，开发速度更快，但团队成员对python几乎零基础，需要重新学习语言，学习成本过高，且python在后期需要大量安装包，环境配置较麻烦

## 后果

- **正面影响**：
  - 类型安全 + 编译期检查减少低级运行时错误
  - Spring Boot 的声明式事务保障订单并发接单时的数据一致性
  - JPA 实体关系映射（User → Order → Review）与业务领域模型天然对齐
- **负面影响**：
  - 相比 Python/Django，Spring Boot 项目初始化配置较多（安全配置、JPA 配置、CORS 配置等），需 3-5 天搭建脚手架
  - Java 代码量多于 Python，同等功能需要的文件数和样板代码更多
  - 启动速度慢于 Django，可能影响开发时的快速试错节奏
- **需要关注的风险**：
  - Spring Boot "约定大于配置"的自动装配可能让初学者困惑——需团队统一理解核心自动配置原理
  - JPA 的懒加载和 N+1 查询问题是常见坑——订单与评价的关联查询需提前设计好 Fetch 策略
  - 需在第 1 周完成 Spring Security + JWT 的认证流程搭建，这是所有业务接口的基础
