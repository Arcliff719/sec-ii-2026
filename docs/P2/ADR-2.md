
# ADR-2：前端语言选择 Vue

## 状态
已接受

## 背景

前端需要构建以下核心页面：用户注册/登录、个人资料管理、需求发布与分类浏览、需求详情、订单列表与状态流转、评价评分、消息通知。总计约 8-10 个页面。

团队前端经验有限，成员无前端开发基础。需要在主流前端框架中选择一个学习成本最低、10 周内可完成 MVP 的方案。

候选方案：Vue 3、React。

## 决策

采用 **Vue 3** 作为前端框架，配合 **Element Plus** 组件库和 **Pinia** 状态管理。

技术栈细节：
- 构建工具：Vite
- UI 组件库：Element Plus（桌面端优先）
- 状态管理：Pinia
- HTTP 客户端：axios
- 路由：Vue Router 4

## 理由

1. **学习曲线最平缓**：Vue 3 的 SFC模板语法接近原生 HTML/CSS/JS，相较于react更易适合无基础团队快速上手。
2. **中文生态完善**：Vue 及 Element Plus 均由国内社区主导，中文文档质量高、更新及时，对中文母语团队友好，减少因语言障碍导致的学习耗时
3. **渐进式设计**：Vue 可渐进式引入——初期简单页面仅用模板+响应式数据即可，复杂页面再引入 Pinia 状态管理和路由守卫，不强制全套生态
4. **Element Plus 组件丰富**：表单、表格、对话框、评分、步骤条等组件直接对应 MVP 的业务需求（需求表单 → 表单组件，订单状态 → 步骤条，评价 → 评分组件），减少 50% 以上的 UI 开发量
5. **为什么不选 React**：React对于零基础团队来说上手较慢，学习成本过高，且生态分散，可选择的内容过多，可能会导致团队陷入“选型困难”

## 后果

- **正面影响**：
  - 开发效率高：Element Plus 组件库覆盖 MVP 90% 的 UI 需求
  - 上手快：团队成员可在 1 周内通过官方中文教程进入开发状态
  - 社区活跃：遇到问题易于搜索到中文解决方案
- **负面影响**：
  - 相比React,Vue的职业生态环境更小，大型企业一般不考虑，可能会对后续就业产生一定影响，但在此课程项目无此问题。
  - Vue 2 → Vue 3 生态仍有部分迁移中的第三方库，选型时需确认 Vue 3 兼容性
  - Element Plus 以桌面端设计为主，移动端体验需额外适配，在后续面对小程序或移动端开发会产生一系列困难
- **需要关注的风险**：
  - 需在第 1 周统一 ESLint + Prettier 配置，避免代码风格混乱
  - 组件拆分粒度需提前规划——页面过多时单文件组件可能膨胀
