## 辩论结果

### 架构风格：分层架构 + 前后端分离

**选择理由**：
- 功能多时方便维护，各层职责清晰，出 bug 不会导致整个程序崩溃
- 测试和维护都能分开，后续测试好搞
- 前后端分离符合项目要求，4 人团队可并行开发

**为什么不选 MVC**：MVC 虽然简单、写代码轻松，但用户量大、功能多时容易变成"石山代码"，后续扩展与测试难以展开。

**为什么不选微服务**：4 人团队 + 10 周周期下学习成本过高，AI 也基本否定了微服务方案。

### 后端语言：Java + Spring Boot

**选择理由**：
- 全体成员课程中学过 Java，有一定基础，后续学习不会太吃力
- Java 是后端主流，项目经历会写进简历，长远来看更有价值
- Java 一次配好后面事少，类型安全适合复杂业务

**为什么不选 Python/Django**：团队 Python 熟练度参差不齐，虽然 Python 语法好学但缺乏项目经验支撑；AI 推荐有 Java 基础的情况下优先选 Java。

### 前端框架：Vue 3

**选择理由**：
- Vue 更像 HTML，学习曲线比 React 平缓，零基础团队友好
- 做普通小平台 Vue 完全够用

**为什么不选 React**：React 偏难，对零基础团队不友好；差距虽不大但 Vue 更简单直接。

### 数据库：MySQL 8.0

**选择理由**：只有这个课程中学过，其他数据库均有额外学习成本，10 周内没时间再学新数据库。

### 测试语言：Python + pytest

**选择理由**：
- Python 搞测试方便
- Java 后端 + Python 测试可通过 HTTP 解耦
- 综合了 Java（后端主流）和 Python（测试高效）各自的优势

### 最终决定

**前端 Vue 3 + 后端 Java（Spring Boot）+ 测试 Python（pytest）+ 数据库 MySQL 8.0**
